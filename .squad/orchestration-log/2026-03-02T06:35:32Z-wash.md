## What was done

Fixed three bugs preventing controller face buttons from working:

1. **Bindings.OnGamePadActiveChanged** — Removed destructive `else` branch clearing all bindings on `isActive=false`. The event fires on every input-source switch (including mouse moves), not just controller events.

2. **ApplyDirectBindings() call sites** — Added `wheelOpen` guards to 7 event handlers to prevent direct spell bindings from clobbering wheel slot bindings.

3. **Frame creation idempotency** — Added existence checks to `Radial:CreateWheelFrames()` and `Radial:InitGamePadButtonHandling()` to guard against duplicate frame creation errors on repeated `OnEnable()` calls.

## Modified files

- `CouchPotato/Core/Bindings.lua` — 7 guards added, destructive else branch removed

## Key insight

`GAME_PAD_ACTIVE_CHANGED` is a high-frequency event unsuitable for state clearing. Real deactivation is covered by `OnCVarUpdate(GamePadEnable=0)` and `OnGamePadDisconnected`. Pattern documented for future development.
