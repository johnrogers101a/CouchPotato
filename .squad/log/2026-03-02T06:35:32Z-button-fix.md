# Session: Controller Face Button Fix

**Date:** 2026-03-02T06:35:32Z  
**Agents:** Wash  
**Tests:** 91/91 passing

## What happened

Wash fixed face button unresponsiveness on controller activation by addressing three root causes:

1. Bindings being cleared on every mouse move (high-frequency `GAME_PAD_ACTIVE_CHANGED` event)
2. Direct spell bindings overwriting wheel slot bindings during wheel-open state
3. Frame creation errors on repeated OnEnable() calls

## Changes

| File | Change |
|------|--------|
| `CouchPotato/Core/Bindings.lua` | Removed destructive else branch; added 6 wheelOpen guards |
| `CouchPotato/Core/GamePad.lua` | Added 1 wheelOpen guard |
| `CouchPotato/UI/Radial.lua` | Added 2 idempotency guards |

## Outcome

Face buttons now work reliably in both direct and wheel modes. Pattern documented for event handler design.
